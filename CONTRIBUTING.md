# Contributing

This is an open-source investigative data journalism project. Contributions that improve the analysis, catch errors, or extend the work are welcome.

## How to contribute

**Found an error in the data or methodology?**
Open an issue. Include the specific claim, the source you believe is more accurate, and a link. We take corrections seriously and will update published analysis.

**Want to reproduce or extend the analysis?**
Each issue folder contains the full pipeline. Clone the repo, run the scripts, and check the numbers. If you find something different, open an issue with your methodology.

**Have a dataset or source we should know about?**
Open an issue or start a Discussion. We're always looking for federal datasets, international comparison data, and peer-reviewed sources that enable original quantitative analysis.

## What we're looking for

- Corrections to published numbers (with sources)
- Alternative methodologies that produce meaningfully different results
- New federal or public datasets relevant to upcoming issues
- Improvements to the Python analysis pipelines
- International comparison data from OECD countries

## What we're not looking for

- Opinion pieces or advocacy content
- Changes to the newsletter prose (that's editorial, not code)
- Pull requests without clear methodology documentation

## Code style

- Python 3.10+
- Scripts should be self-contained and reproducible from a clean clone
- Document data sources and methodology inline
- Include caveats explicitly
